# MyDFIR-SOC-Project

## 🛡️ Live SOC Simulation Project

## 📌 Overview
This project simulates a real-world Digital Forensics and Incident Response (DFIR) and Security Operations Center (SOC) environment using the ELK Stack in a cloud-based lab.

The goal is to develop hands-on experience in:
- Threat detection and analysis
- Log ingestion and correlation
- Attack simulation and investigation
- Incident response and reporting

---

## 🎯 Objectives
- Deploy and configure a SIEM (ELK Stack)
- Ingest endpoint telemetry using Winlogbeat and Sysmon
- Simulate real-world attacks (brute force, malware, C2)
- Create detection rules and dashboards in Kibana
- Perform threat hunting and incident investigations
- Document findings in a structured SOC workflow

---

## 🧰 Tools & Technologies
- Elasticsearch, Logstash, Kibana (ELK Stack)
- Winlogbeat
- Sysmon
- Mythic C2 Framework
- Ubuntu Server (Cloud)
- Windows Endpoint

---

## 🧪 Lab Architecture
This lab consists of:
- Cloud-hosted ELK server for centralized logging
- Windows endpoint generating logs
- Simulated attacker machine for offensive testing

---

## 🔍 Skills Demonstrated
- SIEM deployment and configuration
- Log analysis and event correlation
- Detection engineering
- Threat hunting (C2 activity analysis)
- Incident response lifecycle

---

## 📁 Project Structure

- `01-lab-setup/` → ELK deployment and infrastructure setup  
- `02-log-ingestion/` → Winlogbeat & Sysmon configuration  
- `03-attack-simulation/` → Brute force and malware testing  
- `04-detection-engineering/` → Detection rules & dashboards  
- `05-case management/` → Case management  
- `06-incident-response/` → Incident reports and case handling  
- `screenshots/` → Supporting visuals  

---

## 🚀 Outcome
This project demonstrates end-to-end SOC analyst capabilities, from log ingestion to threat detection, investigation, and incident response using industry-relevant tools and methodologies.
