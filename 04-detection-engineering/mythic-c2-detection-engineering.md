# Mythic C2 Detection Engineering & Threat Monitoring

## 📌 Objective

To create custom detections, alerts, and dashboards capable of identifying malicious activity generated during the Mythic C2 attack simulation.

The goal of this exercise was to:

* Detect execution of the Apollo Mythic agent
* Monitor suspicious process creation activity
* Detect outbound network communications
* Identify Windows Defender tampering
* Build SOC dashboards for threat monitoring
* Generate actionable security alerts

---

# 🏗️ Attack Overview

During the attack simulation, the Apollo Mythic payload was executed on the Windows endpoint.

The payload was renamed:

```text
svchost-tee.exe
```

However, Sysmon telemetry revealed that the original file name remained:

```text
Apollo.exe
```

This provided a reliable detection opportunity.

---

# 🔍 Threat Investigation

Initial investigation began by searching Elasticsearch for:

```text
svchost-tee.exe
```

Reviewing the event timeline revealed multiple Sysmon events associated with the payload execution.

---

# 🧪 Process Creation Analysis

Several Sysmon Process Creation events were identified.

Relevant telemetry included:

| Field                               | Purpose                |
| ----------------------------------- | ---------------------- |
| winlog.event_data.Image             | Executed process       |
| winlog.event_data.CommandLine       | Execution arguments    |
| winlog.event_data.ParentImage       | Parent process         |
| winlog.event_data.ParentCommandLine | Parent process command |
| winlog.event_data.ProcessGuid       | Process correlation    |
| winlog.event_data.User              | Executing user         |
| winlog.event_data.CurrentDirectory  | Execution location     |
| message                             | Event context          |

---

# 🎯 Detection Opportunity

Although the payload filename had been changed, Sysmon preserved the original file metadata.

Observed:

```text
OriginalFileName = Apollo.exe
```

Additionally, the payload generated a unique SHA256 hash.

This allowed creation of high-confidence detections.

---

# 🚨 Apollo Agent Detection Rule

## Detection Logic

The rule was designed to trigger whenever the Apollo agent process was created.

### Detection Query

```kql
event.code: 1 and (
winlog.event_data.Hashes: *8A3BE827182C18001C3CAA8DB1FE0808DBE5A341EB36FFEFA190CA555ACA8A13*
or
winlog.event_data.OriginalFileName: Apollo.exe
)
```

---

## Rule Configuration

| Setting   | Value                   |
| --------- | ----------------------- |
| Rule Name | MyDFIR-Mythic-C2-Apollo |
| Severity  | Critical                |
| Rule Type | Custom Query            |

# 📊 Dashboard Development

A SOC dashboard was created to monitor attacker activity generated by the Mythic simulation.

The dashboard consisted of three primary components.

---

# 🔹 Dashboard 1 – Suspicious Process Creation

Purpose:

Monitor execution of common attacker tooling.

Fields Used:

* User
* Process Name
* Parent Process
* Command Line
* Current Directory

Query:

```kql
event.code: 1 and event.provider: Microsoft-Windows-Sysmon and (powershell or cmd or rundll32)
```

Saved As:

```text
Process Created (PowerShell CMD, Rundll32)
```

---

# 🔹 Dashboard 2 – Outbound Network Connections

Purpose:

Identify processes initiating external communications.

Fields Used:

| Field           |
| --------------- |
| Image           |
| SourceIp        |
| DestinationIp   |
| DestinationPort |

Query:

```kql
event.code: 3 and event.provider: Microsoft-Windows-Sysmon and winlog.event_data.Initiated: true
```

Refined Query:

```kql
event.code: 3 and event.provider: Microsoft-Windows-Sysmon and winlog.event_data.Initiated: true and not winlog.event_data.Image: *MsMpEng.exe and not winlog.event_data.Image: *Traffmonetizer.exe
```

Saved As:

```text
Process Created Initiated Network
```

---

# 🔹 Dashboard 3 – Microsoft Defender Disabled

Purpose:

Detect tampering with endpoint protection.

Relevant Fields:

| Field       |
| ----------- |
| Image       |
| ProductName |
| Event Code  |

Query:

```kql
event.code: 5001 and event.provider: Microsoft-Windows-Windows Defender
```

Saved As:

```text
Microsoft Defender Disabled
```
