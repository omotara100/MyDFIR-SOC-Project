# Windows Server 2022 Deployment

## 📌 Objective
To deploy a Windows Server 2022 instance within the SOC/DFIR lab environment with Remote Desktop Protocol (RDP) exposed to the internet for remote administration and attack simulation scenarios.

This system will later be used to:
- Generate Windows security logs
- Simulate brute-force attacks
- Deploy Sysmon and Winlogbeat
- Monitor endpoint telemetry
- Conduct threat detection and investigation exercises

# 🖥️ Server Deployment

## 🔹 Deployment Steps

1. Created a new Windows Server 2022 instance in Vultr
2. Deployed the server with a public-facing network configuration3. Assigned a public IP address
4. Enabled Remote Desktop Protocol (RDP) access

## 🔓 RDP Exposure

RDP was intentionally exposed to the internet to simulate a realistic attack surface for future SOC investigations and brute-force attack testing.

## 🔐 Security Considerations

The following considerations were noted during deployment:

- RDP exposure increases attack surface visibility
- Firewall rules should be monitored closely
- This configuration is intentionally vulnerable for controlled lab testing
- Future detection rules will monitor RDP authentication attempts and suspicious login activity
- The Windows server was intentionally deployed outside the private VPC network to simulate a publicly exposed enterprise endpoint
- This architecture supports realistic attack simulations targeting internet-accessible RDP services
